# Настройка PyCharm для проекта Trading_bot

## Проблема

Если вы видите ошибку:
```
No Python at 'C:\Users\wangr\AppData\Local\Programs\Python\Python39\python.exe'
```

Это означает, что PyCharm использует виртуальное окружение из другого проекта.

## Решение 1: Настройка интерпретатора в PyCharm (рекомендуется)

1. Откройте PyCharm
2. Перейдите в **File → Settings** (или **Ctrl+Alt+S**)
3. В левом меню выберите **Project: Trading_bot → Python Interpreter**
4. Нажмите на выпадающий список интерпретатора
5. Выберите **Show All...**
6. Нажмите **+** (Add) для добавления нового интерпретатора
7. Выберите **Existing environment**
8. Укажите путь к интерпретатору:
   ```
   C:\Users\wangr\PycharmProjects\Trading_bot\venv\Scripts\python.exe
   ```
9. Нажмите **OK** и примените изменения

## Решение 2: Использование скрипта run.bat

Вместо запуска через PyCharm, используйте скрипт `run.bat`:

1. Дважды кликните на файл `run.bat` в корне проекта
2. Или запустите из командной строки:
   ```cmd
   run.bat
   ```

## Решение 3: Пересоздание виртуального окружения

Если проблема сохраняется, пересоздайте виртуальное окружение:

1. Удалите папку `venv` (или переименуйте в `venv_old`)
2. Откройте терминал в PyCharm (Alt+F12)
3. Выполните команды:
   ```cmd
   python -m venv venv
   venv\Scripts\activate
   pip install -r requirements.txt
   ```

## Проверка правильной настройки

После настройки проверьте:

1. В PyCharm внизу справа должно быть указано: `Python 3.13.3 (venv)`
2. Или выполните в терминале PyCharm:
   ```cmd
   python --version
   ```
   Должно показать: `Python 3.13.3`

## Альтернативный способ запуска

Если PyCharm продолжает использовать неправильный интерпретатор:

1. Откройте терминал в PyCharm (Alt+F12)
2. Активируйте виртуальное окружение:
   ```cmd
   venv\Scripts\activate
   ```
3. Запустите приложение:
   ```cmd
   python main.py
   ```
